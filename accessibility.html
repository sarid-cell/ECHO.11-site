Accessibility Statement

Last updated: March 2026

"Technology should empower everyone. At ECHO.11, we believe that digital experiences must be accessible to all users, regardless of ability or circumstance."

Our Commitment

ECHO.11 is committed to ensuring digital accessibility for people with disabilities. We continually work to improve the user experience for everyone and apply the relevant accessibility standards to achieve this goal.

Conformance Status

We strive to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. These guidelines explain how to make web content more accessible to people with disabilities and more user-friendly for everyone.

Accessibility Features

The following accessibility features have been implemented on this website:

  • Semantic HTML: We use proper heading hierarchy and semantic elements to ensure screen readers can navigate content effectively.
  • Keyboard Navigation: All interactive elements are accessible via keyboard navigation, including skip links to bypass repetitive content.
  • Alt Text: All images include descriptive alternative text to convey information to users who cannot see images.
  • Color Contrast: Text and interactive elements meet WCAG AA contrast ratio requirements for readability.
  • Focus Indicators: Visible focus states are provided for all interactive elements to assist keyboard users.
  • Responsive Design: The website is fully responsive and works across all device sizes and orientations.
  • Reduced Motion: We respect the user's motion preferences and disable animations when the "prefers-reduced-motion" setting is enabled.
  • ARIA Labels: Where appropriate, ARIA labels and roles are used to provide additional context for assistive technologies.
  • Readable Fonts: We use clean, legible typefaces at appropriate sizes for comfortable reading.

Assistive Technology Compatibility

This website is designed to be compatible with the following assistive technologies:

  • Screen readers (such as NVDA, JAWS, VoiceOver, and TalkBack)
  • Screen magnification software
  • Speech recognition software
  • Keyboard-only navigation

Known Limitations

While we strive for comprehensive accessibility, some content may have limitations:

  • Some third-party content (such as embedded social media feeds) may not be fully accessible.
  • Older PDF documents may not be fully accessible; we are working to remediate these.
  • Some interactive elements with complex animations may have reduced functionality when motion is reduced.

Continuous Improvement

We view accessibility as an ongoing effort. We regularly:

  • Conduct accessibility audits of our website
  • Train our content creators on accessibility best practices
  • Review and update our accessibility features based on user feedback
  • Stay informed about evolving accessibility standards and technologies

Technical Specifications

This website relies on the following technologies for accessibility:

  • HTML5
  • CSS3
  • JavaScript
  • WAI-ARIA

These technologies are relied upon for conformance with the accessibility standards used.
